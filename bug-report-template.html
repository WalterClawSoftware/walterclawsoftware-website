Template structure

A useful bug report template keeps facts and gaps in separate boxes.

Use this as a plain starting structure before moving the report into GitHub Issues, Jira, Linear, Markdown, or your internal support workflow.

Summary
One or two sentences naming the visible problem, affected workflow, and user impact as currently understood.
Known facts
Only details already present in the report: account type, feature area, timestamps, input files, error text, screenshots, logs, or customer-provided context.
Steps to reproduce
Numbered repro steps when they are known. If steps are incomplete, label the uncertain step instead of filling it in from a guess.
Expected behavior
What the reporter thought should happen, or the product behavior implied by docs, UI labels, or prior customer expectation.
Actual behavior
What actually happened: error message, wrong output, stalled screen, missing item, crash, or other observable result.
Environment
Operating system, browser, app version, device, permissions, network state, account role, file type, or integration involved.
Evidence
Attach or cite screenshots, logs, console output, ticket IDs, pasted issue text, support notes, and any customer email details the team is allowed to use.
Missing info
Fields that are not known yet, such as exact version, reproducible frequency, last known good behavior, or whether another user can trigger it.
Follow-up questions
Short, answerable questions for support, QA, or the customer before a developer spends time chasing a vague bug.

FAQ

Plain limits before checkout.

These answers keep the template and product positioning honest.

What should a bug report template include?

A practical bug report template should include a short summary, known facts, steps to reproduce, expected behavior, actual behavior, environment, evidence, missing information, and follow-up questions.

Can Repro Pack turn a support ticket into a bug report?

Repro Pack is built for rough bug reports, support notes, customer emails, and copied issue text that the user provides. It helps separate known facts from gaps and draft a cleaner repro packet for handoff.

Does Repro Pack replace Jira, Linear, GitHub Issues, or QA work?

No. Repro Pack is a desktop cleanup utility, not a tracker replacement, QA replacement, customer investigation tool, or real reproduction test. It prepares clearer drafts for tools such as GitHub, Jira, Linear, or Markdown notes.

Will the template guarantee that a developer can reproduce the issue?

No. A clearer template can make known details easier to review, but it cannot guarantee reproduction and should not invent missing evidence. Gaps should stay visible as missing information or follow-up questions.
